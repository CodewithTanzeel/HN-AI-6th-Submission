# Ngrok + ElevenLabs Webhook Integration Guide

This guide explains how to set up ngrok to expose your local backend for ElevenLabs voice agent integration.

## Quick Start

### 1. Install Ngrok

**Windows (using Chocolatey):**
```powershell
choco install ngrok
```

**Windows (manual):**
1. Download from https://ngrok.com/download
2. Extract `ngrok.exe` to a folder in your PATH
3. Run: `ngrok config add-authtoken <your-authtoken>`

**Get your authtoken from:** https://dashboard.ngrok.com/get-started/your-authtoken

### 2. Start the Backend

```bash
# Terminal 1 - Start FastAPI backend
uvicorn app.main:app --app-dir backend --reload
```

### 3. Start Ngrok

```bash
# Terminal 2 - Start ngrok
python scripts/run_ngrok.py
```

This will:
- Start ngrok on port 8000
- Create `frontend/.env.local` with the ngrok URL
- Update `.env` with `NGROK_URL`

### 4. Configure ElevenLabs Agents

In your ElevenLabs dashboard (https://elevenlabs.io/app/conversational-ai), configure your agents:

#### Main Agent: Neogitatera
- **Name:** `Neogitatera`
- **Purpose:** Voice intake for collecting move details
- **Webhook URL:** `https://<your-ngrok-url>/api/webhook/elevenlabs`


## API Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/webhook/elevenlabs` | POST | Receive callbacks from ElevenLabs agents |
| `/api/webhook/call-result` | POST | Receive call results from counterparty agents |
| `/api/webhook/transcript/{job_id}/{vendor_id}` | GET | Retrieve call transcripts |
| `/api/webhook/ngrok-url` | GET | Get current ngrok URL |

## Environment Variables

### Backend (.env)
```bash
# Your ElevenLabs API key (format: xi_...)
ELEVENLABS_API_KEY=xi_your_actual_key_here

# Your OpenAI API key (format: sk_...)
OPENAI_API_KEY=sk_your_openai_key_here

# Ngrok URL (populated by run_ngrok.py)
NGROK_URL=https://your-subdomain.ngrok.io
```

### Frontend (.env.local) - Auto-generated by run_ngrok.py
```bash
NEXT_PUBLIC_API_URL=https://your-subdomain.ngrok.io
NEXT_PUBLIC_ELEVENLABS_API_KEY=xi_your_actual_key_here
```

## Webhook Flow

```
User → Neogitatera Agent (ElevenLabs)
    ↓ (collects move details)
Job Created in Database
    ↓
User → Calls Page → Start Calls
    ↓
Backend → Calls Counterparty Agents
    ↓ (via ElevenLabs)
Carolina/Budget/Premium Agent
    ↓ (calls webhook)
https://<ngrok-url>/api/webhook/call-result
    ↓
Backend stores quote in database
    ↓
Negotiation → Report
```

## Troubleshooting

### Ngrok not starting
- Check if port 8000 is already in use
- Verify ngrok is installed: `ngrok version`
- Make sure you have an authtoken configured

### Webhook not receiving calls
- Verify the ngrok URL is correct in ElevenLabs dashboard
- Check ngrok web interface: http://localhost:4040
- Ensure your backend is running

### CORS errors
- The backend automatically adds ngrok URL to CORS origins
- Restart backend after ngrok starts

### API key issues
- ElevenLabs keys start with `xi_`, not `sk_`
- Make sure the key has Conversational AI access enabled

## Manual Ngrok Setup (Alternative)

If you prefer to run ngrok manually:

```bash
# Start ngrok
ngrok http 8000

# Copy the HTTPS URL and update .env
# Set NGROK_URL in your environment
set NGROK_URL=https://your-url.ngrok.io

# Create frontend .env.local
echo NEXT_PUBLIC_API_URL=https://your-url.ngrok.io > frontend/.env.local
```

## Testing with Mock Mode
